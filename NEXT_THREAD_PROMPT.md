# 새 스레드 시작 프롬프트

이 폴더에서는 기존 노량진 시세 앱과 완전히 분리된 독립형 서비스만 다룹니다.

목표는 `수산시장 소매점용 주문·입금·손질·픽업 관리 SaaS`를 설계하는 것입니다.

단, 고객 노출 관점에서는 단순 운영툴이 아니라 `당일 시세 확인부터 주문, 결제, 진행 조회까지 이어지는 신뢰형 수산 주문 서비스`로 다뤄야 합니다.

전제:

- 오픈카톡은 홍보와 유입 채널로 유지
- 오픈카톡 공지에는 오늘 시세, 품절, 주문 가이드가 계속 올라온다는 점을 고려
- 주문 접수는 외부 웹 주문서로 전환
- 사장님과 직원은 관리자 화면에서 주문 상태를 관리
- 고객 수령 방식은 직접 픽업, 퀵, 택배를 모두 고려
- 실제 운영에서는 최종 금액 확정, 손질비/운임비 반영, 미입금 출고 방지가 중요
- 첫 산출물은 아래 3가지

1. MVP 기능명세서
2. 고객 주문서 + 관리자 화면 구성안
3. 사장님 제안서 문안

문서 방향:

- 현장 운영 문제를 기준으로 기능을 정의
- 과도한 플랫폼 확장보다 빠르게 팔 수 있는 MVP 중심
- 수산시장 소매점 운영 언어로 작성
- 고객가치, 신뢰 형성, 재구매 구조까지 함께 점검
