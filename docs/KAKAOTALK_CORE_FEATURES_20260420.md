# 카카오톡 운영 로그 기반 핵심 기능 재정의

분석 대상:
- `KakaoTalk_20260420_1459_29_942_group.txt`
- `KakaoTalk_20260420_1459_15_725_group.txt`
- `KakaoTalk_20260420_1458_48_669_group.txt`

분석 목적:
- 소비자가 실제로 헷갈리는 지점을 기능으로 바꾼다.
- 운영자가 매일 반복하는 일을 화면 구조로 바꾼다.
- 제안용 데모가 기능 나열이 아니라 실제 운영 문제 해결처럼 보이게 만든다.

## 1. 핵심 결론

이 앱의 핵심은 `수산 쇼핑몰`이 아니다.

정확한 포지션은 다음이다.

> 카톡으로 흩어지는 시세, 주문, 예약금, 반마리, 입금, 수령 안내를 한 화면에서 정리해주는 수산 소매점 운영 시스템

고객에게는 `어떻게 주문해야 하는지 모르겠다`와 `내 주문이 제대로 들어갔는지 모르겠다`를 해결해야 한다.

운영자에게는 `카톡 공지와 문자 주문을 계속 다시 정리해야 하는 문제`를 해결해야 한다.

## 2. 반복 키워드 기준

3개 카톡 로그에서 반복된 주요 키워드는 다음 경향을 보인다.

| 주제 | 반복 의미 |
| --- | --- |
| 주문 | 주문 방법, 접수 여부, 답변 지연, 문자 주문 형식이 반복됨 |
| 반마리 | 한 마리가 부담스러운 고객 수요가 명확함 |
| 예약 | 사전 확보, 예약금, 미입금 취소 규칙이 반복됨 |
| 입금 | 입금 안내 시점, 예약금 여부, 입금 확인 불안이 반복됨 |
| 택배/퀵/방문 | 수령 방식별 마감, 비용, 가능 여부 문의가 많음 |
| 포장/진공/숙성 | 고객은 옵션을 알고 싶지만 세부 조합은 어려워함 |
| 시세/품절 | 오늘 뭐가 있는지, 내일도 있는지, 품절 여부가 핵심 |
| 문자/전화 | 카톡방 공지와 실제 주문 채널이 분리되어 혼선 발생 |

## 3. 소비자 핵심 니즈

### 3.1 주문이 제대로 접수됐는지 알고 싶다

카톡 로그에서 가장 강한 불안은 `주문 문자를 보냈는데 답이 없다`, `입금 안내가 언제 오느냐`, `접수 완료 문자는 받았는데 다음 단계가 없다`이다.

필수 기능:
- 주문 완료 즉시 `주문번호` 발급
- 주문 확인 화면에서 `현재 단계` 표시
- `금액 확인 중 / 입금 대기 / 손질 준비 / 출고 완료` 단계 표시
- 30초 자동 새로고침 또는 수동 새로고침

현재 유지:
- `주문 확인` 페이지
- 주문번호 조회

추가 필요:
- 주문 완료 화면에서 주문번호를 더 크게 표시
- 상태별 고객 안내 문구를 짧게 정리

### 3.2 어떻게 주문해야 하는지 모르겠다

반복 공지에는 항상 주문 예시가 있다.

예:
- 희망 생선
- 한 마리/반마리
- 손질 형태
- 방문 시간
- 이름/연락처

필수 기능:
- 자유 입력보다 버튼 중심 주문서
- 당일 주문은 시세표 품목을 눌러 담기
- 예약 주문은 직접 품목 입력
- 반마리는 별도 복잡한 화면보다 `한 마리 / 반마리` 선택

현재 유지:
- 단순화된 주문서
- 당일 품목 카드 선택
- 예약 직접 입력
- 한 마리/반마리 선택

삭제/비추천:
- 고객 화면에서 포장/숙성/진공을 모두 직접 고르게 하는 방식
- 초보자에게 너무 많은 추천 프리셋 노출

### 3.3 오늘 가능한 품목과 가격을 먼저 알고 싶다

고객은 `도다리 있나요`, `내일도 남아있나요`, `가격이 왜 싼가요`, `품절인가요`를 반복해서 묻는다.

필수 기능:
- 홈에서 오늘 시세 확인
- 주문 화면에서 품목 카드 가격 표시
- 품절/예약문의 상태 표시
- 가격은 kg당 단가와 예상 범위로 표시

현재 유지:
- 시세판
- 카톡 시세표 붙여넣기 자동 추출
- 주문서 품목 카드

추가 필요:
- 품목별 `판매중 / 예약문의 / 품절` 버튼형 수정
- 운영자가 품절 처리를 빠르게 할 수 있는 버튼

### 3.4 수령 방식이 어렵다

택배, 당일택배, 퀵, 방문 픽업은 고객이 스스로 판단하기 어렵다.

필수 기능:
- 수령 방식 선택을 3개로 단순화
- 각 방식의 마감/주의사항 표시
- 주소 입력은 퀵/택배일 때만 필수
- 택배는 회 손질보다 필렛/오로시 추천

현재 유지:
- 픽업/퀵/택배 선택
- 수령 방식별 예상 운임

추가 필요:
- 선택한 수령 방식에 맞는 한 줄 안내
- 주문 완료 후 보관/섭취 안내

### 3.5 반마리와 예약금 구조를 이해하고 싶다

반마리는 반복 수요가 있고, 예약금 미입금 자동취소 규칙도 반복 공지된다.

필수 기능:
- 반마리는 주문서에서 0.5 수량으로 처리
- 반마리 주문은 `매칭 확인 필요` 상태로 운영자에게 표시
- 예약 주문은 예약금 필요 여부를 명확히 안내
- 예약금 미입금 시 취소 가능성을 안내

현재 유지:
- 한 마리/반마리 선택
- 반마리 기본 수량 0.5

추가 필요:
- 관리자 대시보드에 `반마리/예약 확인` 큐 고정
- 반마리 매칭 완료/불가 처리 버튼

## 4. 운영자 핵심 니즈

### 4.1 오늘 시세표를 빨리 올려야 한다

운영자는 이미 카톡 공지를 작성한다. 따라서 새 입력 폼을 다시 배우게 하면 실패 가능성이 높다.

필수 기능:
- 카톡 공지 붙여넣기
- 자동 추출
- 품목별 수정
- 품절/예약문의 빠른 처리
- 오늘 시세 게시

현재 유지:
- 관리자 `오늘 시세 빠른 입력`
- 자동 추출
- 미리보기 수정
- 게시 기능

추가 필요:
- 품목 추가 버튼
- 품절 전환 버튼
- 가격 일괄 수정 편의

### 4.2 주문을 놓치면 안 된다

카톡 로그에는 `바빠서 답변 지연`, `문자 보냈는데 대답 없음`, `돈도 안 받고 물건 보냄` 같은 운영 리스크가 보인다.

필수 기능:
- 오늘 접수 주문 목록
- 미입금 주문 우선 표시
- 준비 시작 주문 표시
- 출고 대기 표시
- 검색

현재 유지:
- 관리자 주문 목록
- 미입금/손질/출고 카드
- 우선 주문 추천

추가 필요:
- 주문 상세에서 상태 변경 버튼을 더 크게 배치
- 미입금 주문은 출고 단계로 못 넘어가게 명확히 차단

### 4.3 예약금과 입금 확인을 분리해야 한다

일부 주문은 예약금, 일부 주문은 최종 금액 입금이다. 고객도 운영자도 이 차이에서 헷갈린다.

필수 기능:
- 예약금 상태
- 최종 금액 상태
- 입금 확인 필요 큐
- 고객 안내 문구 복사

현재 유지:
- 입금 검토 큐
- 고객 안내 문구 복사

추가 필요:
- 예약금 입금/잔금 입금 상태를 분리
- 예약금 미입금 자동취소 예정 표시

### 4.4 카톡 공지를 다시 만들어야 한다

공지방과 대화방이 분리되어 있고, 고객은 공지를 못 보고 질문한다.

필수 기능:
- 시세 반영 후 카톡 공지 자동 생성
- 짧은 공지/상세 공지/예약 강조 공지
- 복사 버튼

현재 유지:
- 카톡 공지 생성기

추가 필요:
- 주문 링크가 포함된 짧은 버전 우선 노출
- 품절 품목은 자동으로 하단 분리

## 5. 유지 / 변경 / 삭제 기준

### 반드시 유지

- 홈
- 주문하기
- 주문 확인
- 운영자 PIN 진입
- 카톡 시세표 붙여넣기
- 오늘 시세 게시
- 주문번호 조회
- 예상 금액 범위
- 반마리 0.5 수량
- 관리자 우선 처리 주문
- 카톡 공지 생성

### 변경해야 함

- 반마리는 별도 고객 탭보다 주문서 내부 선택으로 유지
- 포장/숙성/진공은 고객 직접 선택보다 운영자 추천/메모 중심으로 축소
- 주문 화면은 설명보다 선택 행동 중심으로 유지
- 관리자 화면은 기능 나열보다 오늘 할 일 순서 중심으로 유지

### MVP에서 제거하거나 숨김

- 고객용 반마리 별도 탭
- 손님상/선물용 같은 실제 운영과 안 맞는 프리셋
- 과한 손질 가이드
- 자동 알림톡 실제 연동
- 플랫폼 비교 기능
- 후기/커뮤니티 기능

## 6. 최종 화면 프레임

### 고객 화면

1. 홈
   - 오늘 시세 요약
   - 주문하기 CTA
   - 주문 확인 CTA
   - 마감/수령 방식 짧은 안내

2. 주문하기
   - 오늘 주문 / 예약 주문
   - 한 마리 / 반마리
   - 품목 담기
   - 받는 방법
   - 연락처
   - 예상 금액
   - 주문서 제출

3. 주문 확인
   - 주문번호 입력
   - 현재 단계
   - 입금 안내
   - 다음 행동 안내

### 운영자 화면

1. 오늘 운영 순서
   - 오늘 시세 올리기
   - 먼저 볼 주문
   - 입금/출고 처리
   - 카톡 공지 복사

2. 오늘 시세 빠른 입력
   - 카톡 붙여넣기
   - 자동 추출
   - 품목별 수정
   - 게시

3. 주문 처리
   - 미입금
   - 손질 준비
   - 반마리/예약
   - 출고 대기

4. 안내 문구
   - 고객별 안내 문구 복사
   - 카톡 공지 문구 복사

## 7. 다음 개발 우선순위

1. 주문 완료 화면에서 주문번호 크게 표시
2. 관리자 주문 상세의 상태 변경 버튼 단순화
3. 반마리/예약 큐를 관리자 대시보드에 별도 고정
4. 시세 입력 미리보기에서 품절/예약문의/삭제 버튼 강화
5. 주문 상태별 고객 안내 문구를 실제 운영 톤으로 정리

이 5개가 끝나야 제안용 데모가 `그럴듯한 화면`이 아니라 `실제로 쓰면 편해지는 시스템`으로 보인다.
