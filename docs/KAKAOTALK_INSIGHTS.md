# 카카오톡 대화 기반 현장 요구 분석

## 1. 분석 대상

- 파일: `KakaoTalk_20260401_1014_16_150_group.txt`
- 대화 기간: 2026년 3월 19일 ~ 2026년 3월 30일
- 분석 목적: 실제 오픈카톡 운영에서 반복되는 주문/응대/출고 문제를 제품 기획에 반영

## 2. 핵심 관찰 요약

### 2-1. 주문 접수는 여전히 비정형 개인문자 중심이다

- 공지에서 반복적으로 `주문은 개인문자로 보내주세요`가 안내된다
- 응대 지연을 전제로 운영하고 있어 주문 누락 위험이 있다
- 고객은 어떤 항목을 보내야 하는지 매번 다시 물어본다

### 2-2. 시세표와 품절 정보가 문의량을 크게 좌우한다

- `시세표에 없으면 없는지`, `언제 시세가 올라오는지`, `완판인지`를 반복적으로 묻는다
- 고객은 공지글을 보고도 다시 확인 문의를 남긴다
- 오늘 주문 가능한 품목과 이미 끝난 품목을 구분해서 보여줄 필요가 있다

### 2-3. 수령 방식마다 주문 양식과 운영 규칙이 다르다

- 직접 픽업, 카카오퀵, 일반택배, 당일택배, 고속택배의 요구 정보가 다르다
- 퀵은 `2~3시간 전`, 당일택배는 `평일 오전 9시 30분 전` 같은 마감 규칙이 있다
- 주말 택배 제한처럼 요일별 제약도 존재한다

### 2-4. 손질 방식과 비용 규칙이 상세하다

- 원물, 오로시, 회, 마스카와, 세꼬시 등 가공 방식이 다양하다
- 품목별 추가 손질 비용 규칙이 따로 있다
- 진공포장 가능 여부, 포장비 포함 여부도 고객이 중요하게 본다

### 2-5. 최종 금액 확정과 결제 통제가 중요하다

- `돈도 안받고 물건을 보냈다`, `얼마인지도 모른다`, `영수증을 못 썼다`는 대화가 있었다
- 실제 운영에서는 주문 접수와 최종 금액 확정, 입금 확인이 분리되어 있다
- 출고 전에 결제가 끝났는지 막아주는 장치가 필요하다

### 2-6. 반마리/반절 수요는 반복 수요다

- 여러 사용자가 반반 상대를 구하고, 사장님이 매칭해줄 수 있다는 인식이 있다
- 품목과 크기에 따라 반절 운영 가능 여부가 다르다
- 수령 방식이 다르면 매칭이 깨질 수 있다

### 2-7. 예약 주문과 내일 물건 확보 수요가 있다

- 다음날 시세로 예약하거나 `경매 때 잡아달라`는 요청이 나온다
- 연어처럼 시간 제한이 있는 예약 품목도 있다
- 즉시 주문과 예약 주문은 관리 흐름을 분리해야 한다

### 2-8. 결제 수단 안내 수요도 있다

- 계좌이체 외에 디지털 온누리, 제로페이 바코드 문의가 있다
- 전부 자동 연동할 필요는 없지만 고객 안내와 관리자 메모는 필요하다

## 3. 제품에 반영해야 할 요구사항

### P0. 바로 반영해야 하는 항목

- 고객용 `오늘 시세/주문 가이드` 화면
- 품목별 품절 상태와 주문 가능 여부 표시
- 수령 방식별 주문 양식 분기
- 수령 방식과 가공 방식 조합에 대한 경고 또는 제한
- 최종 금액 확정 후 입금 요청 흐름
- 출고 전 미입금 차단 또는 강한 경고
- 손질 옵션/비용표 관리
- 예약 주문 관리

### P1. MVP 직후 우선 항목

- 반절 주문 매칭 보드
- 입금 검토 필요 큐
- 품절/완판 상태를 공지용으로 쉽게 전환하는 기능
- 결제 수단 안내와 대체 결제 처리 메모

### P2. 이후 고도화 항목

- 자동 입금 대조
- 시세표/공지 자동 생성 보조
- 오픈카톡 유입과 주문서 전환 성과 추적

## 4. 제품 방향에 주는 의미

이 서비스는 단순한 주문서가 아니라 아래 세 가지를 같이 해결해야 한다.

1. 오픈카톡 공지 기반의 `시세/정책 안내`
2. 비정형 개인문자를 대체하는 `구조화된 주문 접수`
3. 출고 전 결제와 수령 방식까지 묶는 `운영 통제`
