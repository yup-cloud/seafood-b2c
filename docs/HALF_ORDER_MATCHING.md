# 반절 주문 매칭 기능 상세 설계

## 1. 기능 목적

원물 전체 구매가 부담스러운 고객도 받을 수 있게 하면서, 매장은 원물 단위 판매 구조를 유지할 수 있도록 `반절 구매 희망 주문`을 서로 매칭해주는 기능이다.

이 기능의 핵심 가치는 두 가지다.

- 고객 측면: 한 마리가 부담스러운 1인 가구, 2인 가구, 소규모 모임 수요를 흡수
- 매장 측면: 원물 한 마리 판매 기회를 놓치지 않으면서 소량 수요를 체계적으로 운영

## 2. 기본 개념

### 2-1. 주문 유형

- 전체 주문: 원물 전체 구매를 원하는 주문
- 반절 주문: 원물 반절 구매를 희망하는 주문

### 2-2. 매칭 단위

- 반절 주문 2건을 하나의 `매칭 그룹`으로 묶는다
- 매칭 그룹이 만들어지면 하나의 원물 판매가 성립한 것으로 본다

### 2-3. 핵심 상태값

- 접수 완료
- 매칭 대기
- 매칭 후보 있음
- 매칭 완료
- 입금 대기
- 입금 완료
- 작업 진행
- 매칭 실패
- 취소

## 3. 고객 주문 흐름

1. 고객이 주문서에서 품목, 규격, 수량, 손질 방식, 수령 시간을 입력한다.
2. 고객이 구매 단위에서 `반절 구매 희망`을 선택한다.
3. 주문은 곧바로 확정되지 않고 `매칭 대기` 상태로 저장된다.
4. 시스템은 같은 품목과 유사 조건의 반절 주문을 탐색한다.
5. 조건이 맞는 주문이 있으면 관리자에게 추천하거나 자동으로 매칭 그룹을 만든다.
6. 매칭이 완료되면 각 고객에게 `주문 확정 및 입금 요청` 안내를 보낸다.
7. 두 주문 모두 입금이 확인되면 일반 주문과 같은 작업 흐름으로 넘어간다.

## 4. 관리자 운영 흐름

### 4-1. 반절 매칭 보드

관리자는 `반절 매칭 관리 화면`에서 현재 대기 중인 주문을 한 번에 본다.

기본 표시 항목:

- 주문번호
- 주문자명
- 품목명
- 규격
- 희망 수령 날짜
- 희망 수령 시간대
- 손질 방식
- 매칭 대기 시간
- 추천 후보 수

### 4-2. 관리자 액션

- 자동 추천된 후보 확인
- 수동으로 두 주문 연결
- 조건이 애매한 주문은 보류
- 매칭 실패 시 고객 안내 상태로 전환
- 전체 구매 전환 제안 메모 기록

## 5. MVP 기준 추천 운영 방식

첫 버전은 `완전 자동 매칭`보다 `추천 + 관리자 최종 확인` 구조가 안전하다.

추천 이유:

- 규격이 비슷해도 실제 현장에서는 손질 방식 차이, 수령 시간 차이, 당일 물량 차이가 있음
- 고객 기대치가 달라 자동으로 바로 묶으면 분쟁이 생길 수 있음
- 운영 초반에는 매칭 기준을 현장에서 배우는 과정이 필요함

따라서 1단계는 아래와 같이 제안한다.

1. 시스템이 유사 주문을 자동 추천
2. 관리자가 실제 가능 여부를 확인
3. 관리자가 `매칭 확정` 버튼을 눌러 두 주문을 연결
4. 고객에게 입금 요청 및 확정 메시지 발송

## 6. 매칭 조건 설계

### 6-1. 필수 일치 조건

- 같은 품목
- 같은 수령 날짜
- 같은 시간대 또는 허용 오차 범위 내 수령 시간
- 매장에서 허용하는 유사 규격 범위 내 주문
- 같은 수령 방식 또는 함께 처리 가능한 수령 방식 조합

### 6-2. 추천 점수에 반영할 조건

- 손질 방식 유사도
- 포장 방식 유사도
- 주문 접수 시점 차이
- 단골 여부 또는 우선 고객 여부

### 6-3. 운영 규칙으로 반드시 정해야 할 항목

- 매칭 대기 허용 시간
- 같은 시간대로 볼 기준
- 규격 차이 허용 범위
- 반절 가격 계산 방식
- 손질 방식이 서로 다를 때 허용 여부
- 택배 불가일 또는 택배 마감 시간과의 충돌 처리
- 품목/사이즈별 반절 허용 최소 기준

## 7. 가격 정책 초안

첫 버전에서는 계산이 복잡해지지 않도록 단순한 규칙이 필요하다.

권장안:

- 반절 주문은 기본적으로 `전체 예상 판매가의 50%`를 기준으로 시작
- 손질 추가 비용은 주문별로 개별 부과
- 특수 옵션이 붙는 경우 관리자가 최종 금액을 확정

예외적으로 고려할 항목:

- 한쪽만 고급 손질을 요청한 경우
- 한쪽만 추가 포장 옵션을 넣은 경우
- 규격 차이가 큰 반절끼리 묶는 경우
- 수령 방식이 달라 운임 정책이 달라지는 경우

## 8. 고객 안내 문구 방향

### 8-1. 주문서 안내

반절 구매는 같은 조건의 주문이 연결되면 최종 확정됩니다. 매칭 완료 후 입금 안내가 전달됩니다.

### 8-2. 매칭 완료 안내

요청하신 반절 주문이 연결되었습니다. 안내드린 금액으로 입금해주시면 준비가 시작됩니다.

### 8-3. 매칭 실패 안내

현재 같은 조건의 반절 주문이 연결되지 않아 이번 주문은 자동 확정되지 않았습니다. 원물 전체 구매 또는 다음 입고 일정으로 다시 안내드릴 수 있습니다.

## 9. 예외 상황 처리

### 9-1. 한쪽만 입금한 경우

- 다른 한쪽의 입금 마감 시간까지 대기
- 마감 시간 초과 시 관리자 검토 상태로 전환
- 선택지:
  - 이미 입금한 고객에게 대체 제안
  - 전체 구매 전환 제안
  - 환불 또는 취소

### 9-2. 한쪽이 취소한 경우

- 남은 주문을 다시 `매칭 대기`로 복귀
- 일정 시간 안에 새 후보가 없으면 고객에게 안내

### 9-3. 손질 조건이 충돌하는 경우

- 관리자 수동 승인 없이 자동 매칭하지 않음
- 충돌 기준은 운영 초기에 엄격하게 잡는 것이 안전함

### 9-4. 시간대가 너무 다른 경우

- 같은 날이라도 준비 흐름이 엇갈리면 자동 매칭 제외

### 9-5. 수령 방식이 충돌하는 경우

- 주말 택배 불가일에 택배 주문과 픽업 주문을 무리하게 묶지 않음
- 회 + 일반택배처럼 품질 위험이 큰 조합은 관리자 승인 없이 확정하지 않음

## 10. 화면 설계 포인트

### 10-1. 고객 주문서

- 구매 단위 선택값 추가
- 반절 주문 안내 문구 표시
- 매칭 확정 후 입금한다는 점을 명확히 안내

### 10-2. 관리자 대시보드

- `매칭 대기` 카드 추가
- `매칭 실패 위험` 카드 추가 가능

### 10-3. 반절 매칭 관리 화면

- 유사 주문 추천 리스트
- 예상 적합도 점수
- 연결 버튼
- 보류 버튼
- 실패 처리 버튼

## 12. 개발 우선순위 제안

### 1단계

- 주문서에 반절 선택값 추가
- `매칭 대기` 상태 추가
- 관리자 수동 매칭 화면 구현
- 품목/사이즈별 매칭 가능 기준 설정

### 2단계

- 조건 기반 추천 로직 추가
- 매칭 완료 후 자동 안내 메시지 연결

### 3단계

- 자동 매칭 룰 고도화
- 수요가 많은 품목별 매칭 추천 최적화

## 13. 성공 판단 기준

- 반절 주문 문의를 실제 주문으로 전환하는 비율이 올라감
- 매칭 실패율이 초기 운영 기준 이하로 유지됨
- 매칭 관련 고객 문의나 분쟁이 과도하게 발생하지 않음
- 전체 판매 기회 대비 소량 고객 흡수율이 높아짐
