# 사장님 제안서 문안

## 1. 제안 한 줄 요약

오픈카톡 주문은 그대로 두고, 주문 접수부터 입금 확인, 손질, 직접 픽업, 퀵, 택배 관리만 따로 정리해주는 소매점 전용 운영 도구입니다.

## 2. 현재 문제 요약

사장님들이 가장 많이 겪는 문제는 주문이 없는 게 아니라, 주문이 들어와도 정리가 안 된다는 점입니다.

- 카톡으로 주문이 들어오면 품목, 수량, 손질 방식, 픽업 시간이 제각각이라 다시 확인해야 합니다.
- 입금자명과 주문자명이 달라 입금 확인이 늦어집니다.
- 손질 요청이 카톡 대화 사이에 섞여서 직원이 헷갈립니다.
- 누가 어떤 주문을 받았는지 공유가 안 되면 누락이나 중복 응대가 생깁니다.
- 바쁜 시간에는 전화로 주문 상태를 다시 설명하는 것만으로도 일이 밀립니다.
- 원물 전체는 부담스럽고 반절만 원하는 손님은 받아주기 어려워 판매 기회를 놓치기도 합니다.
- 직접 픽업, 퀵, 택배 주문이 섞이면 주소 확인, 인계 여부, 발송 여부를 따로 기억해야 해서 더 꼬이기 쉽습니다.
- 시세표에 없는 품목이 정말 없는지, 오늘 택배가 되는지, 반마리가 가능한지 같은 질문이 반복돼 응대 피로가 쌓입니다.
- 손질비, 운임비까지 포함한 최종 금액 확정이 늦어지면 결제 누락 위험도 생깁니다.

결국 문제는 홍보가 아니라 운영 정리입니다. 손님은 있는데 주문 흐름이 정리되지 않아 시간과 신뢰가 같이 새고 있습니다.

## 3. 서비스 소개

이 서비스는 수산시장 소매점이 바로 사용할 수 있도록 만든 주문 운영 SaaS입니다.

핵심 방식은 단순합니다.

1. 손님은 오픈카톡으로 문의합니다.
2. 매장은 오늘 시세와 주문 가이드를 보여줍니다.
3. 손님은 웹 주문서에 주문 내용을 입력합니다.
4. 매장은 최종 금액을 확정하고 입금 확인, 손질, 직접 픽업, 퀵, 택배 상태를 관리합니다.

즉, 카톡은 유입 채널로 유지하고 주문 확정과 운영 관리는 따로 정리해주는 구조입니다.

추가로 확장하면 이런 기능도 붙일 수 있습니다.

- 반절만 필요한 손님끼리 주문을 묶어주는 매칭 기능
- 계좌 입금 내역을 자동 대조해 입금확인을 줄여주는 기능

## 4. 사장님 입장에서 달라지는 점

### 주문 받을 때

- 시세표, 품절, 예약 가능 여부를 더 명확하게 안내할 수 있습니다.
- 손님마다 다른 말투로 주문해도 최종 접수 형식은 같아집니다.
- 빠지는 정보 없이 주문을 받을 수 있습니다.
- 직원이 대신 받아도 형식이 같아서 인수인계가 쉬워집니다.

### 입금 확인할 때

- 최종 금액을 먼저 확정한 뒤 입금 요청을 보낼 수 있습니다.
- 미입금 주문만 따로 볼 수 있습니다.
- 입금 확인이 끝난 주문만 다음 작업으로 넘길 수 있습니다.
- 누가 언제 확인했는지 기록이 남습니다.

### 손질할 때

- 어떤 주문을 먼저 준비해야 하는지 바로 보입니다.
- 손질 방식과 요청사항이 주문별로 정리됩니다.
- 픽업 시간 기준으로 작업 순서를 맞출 수 있습니다.
- 반절 주문이 붙으면 한 마리 판매 기회를 더 자연스럽게 만들 수 있습니다.

### 출고할 때

- 직접 픽업, 퀵, 택배 주문을 따로 분리해서 볼 수 있습니다.
- 퀵 기사에게 넘겼는지, 택배 송장을 입력했는지 기록으로 남길 수 있습니다.
- 포장 완료 후 어떤 주문이 아직 매장에 남아 있는지 바로 보입니다.

### 손님 응대할 때

- 전화가 오면 주문 상태를 바로 확인할 수 있습니다.
- 주문 누락이나 헷갈림이 줄어 신뢰도가 올라갑니다.
- 입금확인이 자동화되면 계좌 확인 때문에 손이 끊기는 일이 줄어듭니다.

## 5. 기대 효과

- 주문 누락 감소
- 입금 확인 시간 단축
- 손질 순서 혼선 감소
- 사장님 직접 확인해야 하는 건수 감소
- 직원 교육 시간 단축
- 반복 문의 감소
- 반마리/반절 수요 흡수로 추가 매출 기회 확보
- 픽업/퀵/택배 출고 혼선 감소
- 시세표/주문 가능 여부 문의 감소
- 미정산 출고 위험 감소

이 서비스는 매출 분석 툴이 아니라, 바쁜 시간에 덜 꼬이게 만드는 운영 도구입니다.

## 6. 도입 방식

### 1단계. 매장용 주문서 세팅

- 매장명, 계좌번호, 기본 주문 항목, 손질 옵션을 맞춰 세팅합니다.

### 2단계. 관리자 화면 오픈

- 사장님과 직원이 휴대폰 또는 태블릿에서 주문 상태를 확인할 수 있게 설정합니다.

### 3단계. 오픈카톡 응대 문구 연결

- 기존 카톡 응대 문구에 주문서 링크만 붙이면 바로 운영 가능합니다.

### 4단계. 현장 테스트

- 실제 주문 5건에서 10건 정도로 흐름을 점검하고 항목을 보정합니다.

## 7. 가격 모델 초안

### 기본안

- 초기 세팅비: 300,000원
- 월 사용료: 99,000원

### 포함 범위

- 고객 주문서 1종
- 관리자 화면 1개
- 기본 상태 관리
- 입금 확인 관리
- 손질/픽업/퀵/택배 운영 관리

### 별도 협의 가능 항목

- 다점포 운영
- 배송 전용 프로세스 추가
- 품목별 복잡한 옵션 확장
- 맞춤 문자/알림 기능
- 반절 주문 매칭 기능
- 자동 입금확인 연동

## 8. 사장님 설명용 짧은 멘트

사장님, 손님은 계속 카톡으로 들어오게 두셔도 됩니다. 다만 주문 확정만 링크로 받으면 품목, 손질, 입금, 픽업이 한 번에 정리돼서 직원들이 훨씬 덜 헷갈립니다. 결국 주문을 더 많이 받는 것보다, 들어온 주문을 안 놓치고 빨리 처리하는 데 돈을 쓰는 구조입니다.

직접 픽업뿐 아니라 퀵과 택배까지 한 화면에서 나눠서 관리하면, 포장 이후 출고 단계에서 생기는 혼선도 같이 줄일 수 있습니다.

여기에 반절 주문 매칭이나 자동 입금확인까지 붙이면, 지금 놓치던 소량 수요까지 받고 사장님이 직접 계좌를 계속 확인하는 시간도 줄일 수 있습니다.

## 9. 조금 더 직접적인 영업 멘트

지금은 주문이 카톡방 안에 쌓여서 사람이 기억으로 운영하는 구조입니다. 이걸 주문서와 관리자 화면으로 바꾸면 주문 누락, 미입금 혼선, 손질 실수가 줄어듭니다. 큰 플랫폼처럼 복잡하게 가는 게 아니라, 사장님 가게에서 오늘 바로 쓸 수 있는 수준으로 맞춰드리는 게 목표입니다.

특히 반절만 필요한 손님을 연결해주는 기능은 다른 매장과 차별화 포인트가 될 수 있고, 자동 입금확인은 사장님 시간이 가장 많이 들어가는 확인 업무를 줄여주는 기능이 됩니다.

## 10. 마무리 제안 문구

이 서비스는 손님을 새로 모으는 광고 상품이 아니라, 이미 들어오는 주문을 더 정확하고 빠르게 처리하게 만드는 운영 상품입니다. 오픈카톡 중심 영업은 그대로 유지하면서, 주문 확정 이후의 혼선을 줄이고 싶은 매장에 가장 적합합니다.
