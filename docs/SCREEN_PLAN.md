# 화면 구성안

## 1. 화면 구성 목표

첫 화면 구성의 목적은 예쁜 쇼핑몰을 만드는 것이 아니라, 주문 누락과 작업 혼선을 줄이는 것이다. 고객은 빠르게 입력하고, 매장은 빠르게 확인하고, 직원은 바로 작업 순서를 이해할 수 있어야 한다.

## 2. 고객 주문서 화면

### 2-1. 화면 목적

- 카톡 대화 대신 주문 정보를 표준 형식으로 받기
- 주문에 필요한 필수 정보를 한 번에 입력받기
- 손질, 픽업, 입금 정보를 누락 없이 확보하기

### 2-2. 화면 구성

#### A. 상단 안내 영역

- 매장명
- 간단 소개 문구
- 주문 전 안내 문구
- 오늘 주문 가능 품목 요약
- 품절 또는 완판 품목 배지
- 예시:
  - 원하는 품목, 손질 방식, 픽업 시간을 입력해주시면 확인 후 순서대로 준비해드립니다.

#### B. 주문자 정보

- 주문자명
- 연락처
- 입금자명

#### C. 주문 내용

- 품목명
- 오늘 주문 가능 품목 선택 또는 직접 입력
- 규격 또는 사이즈
- 수량
- 구매 단위 선택
  - 원물 전체
  - 반절 구매 희망
- 손질 방식 선택
  - 예: 회손질, 매운탕용 손질, 포장만, 손질 없음
- 손질비 안내
- 추가 요청사항

#### D. 수령 정보

- 수령 방식
  - 직접 픽업
  - 퀵
  - 택배
- 희망 수령 날짜
- 희망 수령 시간
- 직접 픽업일 경우 방문 예정 시간 안내
- 퀵일 경우 수령인명, 연락처, 주소, 퀵 요청 메모 입력
- 택배일 경우 수령인명, 연락처, 주소, 우편번호 입력

#### E. 입금 안내

- 입금 계좌 안내 문구
- 입금 전 확인 문구
- 주문 제출 후 입금 확인 순서 안내
- 반절 주문 선택 시 매칭 완료 후 최종 확정 및 입금 안내가 나간다는 설명
- 최종 금액이 주문 직후 확정되지 않을 수 있다는 안내
- 택배/퀵 운임비가 별도로 붙을 수 있다는 안내

#### F. 제출 버튼

- 버튼 문구 예시:
  - 주문 접수하기

### 2-3. 고객 주문서 UX 원칙

- 한 화면 길이는 길어도 괜찮지만 항목은 단순해야 함
- 필수값과 선택값을 명확히 구분함
- 손질 방식은 텍스트 입력보다 선택형이 우선
- 모바일 카카오톡 브라우저에서 바로 입력 가능해야 함

## 3. 주문 완료 화면

### 3-1. 화면 목적

- 주문이 정상 접수됐다는 확신 제공
- 고객이 다음 행동을 헷갈리지 않게 안내

### 3-2. 포함 요소

- 주문 접수 완료 메시지
- 주문자명 또는 주문번호
- 입금 안내 재노출
- 확인 연락 기준 안내
- 문의용 오픈카톡 링크 또는 연락처
- 반절 주문일 경우 `현재 매칭 대기 중` 상태 문구
- 직접 픽업 / 퀵 / 택배에 따라 다음 단계 안내 문구 분기
- `최종 금액 확정 후 입금 안내 예정` 상태 문구

### 3-3. 예시 문구

주문이 접수되었습니다. 입금 확인 후 준비가 시작됩니다. 요청하신 수령 시간에 맞춰 준비할 수 있도록 입금자명을 정확히 맞춰주세요.

## 4. 관리자 대시보드

### 4-1. 화면 목적

- 사장님이 오늘 주문 상황을 첫 화면에서 바로 이해하기
- 어떤 주문이 막혀 있는지 즉시 파악하기

### 4-2. 상단 요약 카드

- 오늘 신규 주문
- 최종 금액 확정 대기
- 미입금 주문
- 손질 대기
- 오늘 픽업 예정
- 완료 건수
- 반절 매칭 대기
- 입금 검토 필요

### 4-3. 메인 주문 리스트

- 주문번호
- 주문자명
- 연락처
- 주요 품목 요약
- 손질 방식 요약
- 수령 방식
- 수령 시간
- 입금 상태
- 주문 상태
- 출고 상태
- 메모 여부

### 4-4. 기본 필터

- 오늘 주문
- 미입금
- 자동 입금확인 완료
- 입금 검토 필요
- 손질 대기
- 포장 완료
- 직접 픽업 대기
- 퀵 출고 대기
- 택배 발송 대기
- 취소
- 매칭 대기

### 4-5. 빠른 액션

- 최종 금액 입력
- 입금 확인
- 상태 변경
- 상세 보기
- 전화번호 확인
- 매칭 연결

## 5. 주문 상세 화면

### 5-1. 화면 목적

- 전화 문의나 현장 확인 시 주문 한 건을 끝까지 처리하는 화면

### 5-2. 화면 구성

#### A. 상단 고정 정보

- 주문번호
- 현재 상태
- 주문자명 / 연락처
- 수령 방식
- 희망 수령 시간
- 구매 단위
- 반절 주문일 경우 매칭 상태

#### B. 주문 내용 영역

- 품목별 상세
- 규격/수량
- 손질 옵션
- 포장 관련 요청

#### C. 입금 정보 영역

- 최종 금액
- 손질비 / 운임비 요약
- 입금자명
- 입금 상태
- 입금 확인 버튼
- 확인 시간 / 확인 담당자
- 자동 매칭 결과
  - 자동 확인 완료
  - 검토 필요
  - 미입금

#### D. 수령/출고 정보 영역

- 직접 픽업 / 퀵 / 택배 구분
- 수령인명 / 연락처
- 주소
- 퀵 전달 메모
- 택배 송장번호
- 출고 상태

#### E. 작업 메모 영역

- 고객 요청사항
- 내부 메모
- 주의 표시

#### F. 상태 변경 영역

- 다음 단계로 변경
- 취소 처리
- 완료 처리

#### G. 이력 영역

- 주문 생성 시간
- 최종 금액 확정 시간
- 상태 변경 시간
- 메모 작성 이력
- 자동 입금 매칭 로그
- 반절 매칭 연결 이력

## 6. 반절 매칭 관리 화면

### 6-1. 화면 목적

- 반절 구매 희망 주문을 모아서 어떤 주문끼리 묶을지 판단하는 화면

### 6-2. 리스트 항목

- 품목명
- 규격
- 수량
- 희망 수령 시간
- 손질 방식
- 현재 매칭 상태

### 6-3. 주요 액션

- 유사 주문 추천 보기
- 두 주문 연결
- 매칭 실패 처리
- 전체 구매 제안 메모 남기기

## 7. 작업 큐 화면

### 7-1. 화면 목적

- 직원이 지금 바로 어떤 주문부터 손질해야 하는지 판단하는 화면

### 7-2. 정렬 기준

- 희망 수령 시간 빠른 순
- 미입금 주문 제외 또는 별도 표시
- 긴급 표시 주문 상단 노출

### 7-3. 카드 항목

- 주문자명
- 품목 요약
- 손질 방식
- 픽업 시간
- 특이 요청
- 현재 상태

### 7-4. 작업 중심 액션

- 손질 시작
- 손질 완료
- 포장 완료

## 8. 출고 관리 화면

### 8-1. 화면 목적

- 포장 완료 이후 직접 픽업, 퀵, 택배 주문을 방식별로 분리해서 관리하는 화면

### 8-2. 구분 탭

- 직접 픽업 대기
- 퀵 출고 대기
- 택배 발송 대기

### 8-3. 카드 항목

- 주문자명
- 수령 방식
- 수령인명
- 주소 또는 픽업 시간
- 포장 완료 시간
- 메모 여부

### 8-4. 주요 액션

- 직접 픽업 완료 처리
- 퀵 인계 완료 처리
- 택배 송장 입력
- 택배 발송 완료 처리
- 미입금 출고 경고 확인

## 9. 모바일 관리자 사용 흐름

### 9-1. 사용 상황

- 사장님이 매장 안에서 휴대폰으로 주문 확인
- 직원이 작업대 근처에서 상태만 빠르게 변경
- 외부 이동 중에도 미입금 여부 확인

### 9-2. 모바일 우선 설계 포인트

- 리스트에서 핵심 정보만 먼저 보이게 함
- 상태 변경 버튼을 손가락으로 누르기 쉽게 크게 배치함
- 상세 화면에서 전화번호와 메모를 바로 볼 수 있게 함
- 긴 입력보다 선택형 버튼을 우선 사용함
- 자동 입금확인 결과와 매칭 대기 여부를 색상이나 배지로 바로 보이게 함
- 퀵 기사 연락처, 택배 주소, 송장 입력이 모바일에서도 가능해야 함

## 10. 추천 화면 흐름

1. 고객이 주문서 작성
2. 주문 완료 화면 확인
3. 반절 주문이면 매칭 대기 또는 매칭 연결
4. 관리자 대시보드에서 신규 주문 확인
5. 자동 또는 수동으로 입금 확인
6. 주문 상세에서 상태 확인
7. 작업 큐에서 손질 진행
8. 포장 완료 후 출고 관리 화면으로 이동
9. 직접 픽업 / 퀵 / 택배 방식에 따라 완료 처리

## 11. MVP 화면 우선순위

### 반드시 필요한 화면

- 오늘 시세/주문 가이드 화면
- 고객 주문서
- 주문 완료 화면
- 관리자 대시보드
- 주문 상세 화면
- 작업 큐 화면
- 출고 관리 화면

### 있으면 좋은 화면

- 시세/품절 관리 화면
- 예약 주문 관리 화면
- 정산 검토 화면
- 반절 매칭 관리 화면
- 주간 통계 화면
- 단골 고객 목록
- 배송 전용 보드

## 12. 오늘 시세/주문 가이드 화면

### 12-1. 화면 목적

- 오픈카톡 공지에서 반복되는 질문을 줄이고 고객이 주문 가능 여부를 스스로 판단하게 만드는 화면

### 12-2. 포함 요소

- 오늘 시세표
- 품목별 규격/가격
- 품절 또는 완판 상태
- 예약 가능 품목 표시
- 직접 픽업 / 퀵 / 택배 주문 조건
- 회/오로시/원물별 권장 수령 방식
- 영업시간, 주소, 계좌, 문의 링크

### 12-3. 주요 액션

- 주문서로 이동
- 품절 품목 숨김 또는 표시 전환
- 예약 주문 안내 보기

## 13. 정산 검토 화면

### 13-1. 화면 목적

- 최종 금액 확정 전 주문과 입금 전 출고 위험 주문을 한 번에 보는 화면

### 13-2. 리스트 항목

- 주문번호
- 주문자명
- 품목/규격
- 손질비
- 운임비
- 최종 금액 상태
- 입금 상태
- 출고 위험 여부

### 13-3. 주요 액션

- 최종 금액 입력
- 입금 요청 상태로 변경
- 미입금 출고 예외 승인
- 정산 메모 남기기
