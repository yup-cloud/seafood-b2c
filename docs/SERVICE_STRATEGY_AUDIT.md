# 서비스 구조화 및 문제점 진단

## 1. 서비스 한 줄 재정의

오늘바다는 `오픈카톡으로 유입된 수산 주문을 더 쉽고 더 믿을 수 있게 완성시키는 주문 서비스`다.

운영자 입장에서는 주문 누락과 입금 확인, 손질 순서, 출고 관리를 줄여주는 운영 도구이고, 고객 입장에서는 `시세 확인`, `쉬운 주문`, `진행 상황 확인`, `안심 결제`를 한 번에 해결해주는 구매 경험이어야 한다.

## 2. 지금 서비스의 본질

현재 구조는 크게 네 단계로 나뉜다.

1. `유입`
   - 오픈카톡과 공지에서 고객이 오늘 시세와 주문 링크를 확인한다.
2. `전환`
   - 고객이 웹 주문서에서 품목, 손질, 수령 방법, 포장 옵션을 입력한다.
3. `운영`
   - 매장이 최종 금액 확정, 입금 확인, 손질, 포장, 출고를 처리한다.
4. `재구매`
   - 고객이 주문 상태를 확인하고 다음 주문으로 다시 돌아온다.

현재 서비스는 `운영 구조`는 비교적 잘 잡혀 있지만, `고객 신뢰 형성`과 `구매 전환`을 더 강하게 설계할 여지가 있다.

## 3. 핵심 고객 세그먼트

### 3-1. 처음 주문하는 고객

- 용어가 어렵고 손질/포장/숙성지 차이를 잘 모른다.
- 지금 주문해도 되는지, 얼마쯤 나오는지, 어떤 방식으로 받아야 하는지가 가장 중요하다.

### 3-2. 재주문 고객

- 이전과 비슷한 조건으로 더 빨리 주문하고 싶다.
- 새로 설명하는 과정 없이 익숙한 흐름을 원한다.

### 3-3. 예약 구매 고객

- 내일 경매 물건이나 특정 사이즈를 미리 확보하고 싶다.
- 예약금, 확정 시점, 남은 결제 금액이 명확해야 안심한다.

### 3-4. 선물/가족 모임 고객

- 포장 상태, 전달 시간, 신선도 유지 방식이 중요하다.
- 가격보다 실수 없는 전달 경험과 체면이 더 중요하다.

## 4. 현재 서비스의 강점

- 당일 시세를 먼저 보여주는 구조라 가격 신뢰를 만들기 좋다.
- 수령 방식, 손질, 포장, 반마리 주문까지 현장 운영 요구가 잘 반영되어 있다.
- 고객 주문 후 상태 조회가 가능해 일반 카톡 주문보다 훨씬 체계적이다.
- 관리자 관점에서 주문, 입금, 손질, 출고가 한 흐름으로 이어진다.

## 5. 현재 서비스의 문제점

### 5-1. 고객이 느끼는 브랜드 가치가 아직 충분히 선명하지 않다

- 서비스 설명이 여전히 `주문 관리 도구` 관점으로 보일 수 있다.
- 고객은 기능보다 `왜 믿고 주문해도 되는지`를 먼저 본다.
- 첫 화면에서 신뢰 근거와 차별점이 더 분명해야 한다.

### 5-2. 주문 전환 직전의 불안 해소 장치가 부족하다

- 예상 금액 범위가 약하게 느껴질 수 있다.
- 오늘 주문 마감 시간과 수령 방식별 제한이 더 선명하게 보여야 한다.
- 초보 고객은 손질, 숙성지, 포장 조합을 여전히 어렵게 느낄 가능성이 있다.

### 5-3. 주문 후 안심 경험이 아직 약하다

- 조회 토큰 입력은 운영상 가능하지만 고객 입장에서는 한 번 더 생각해야 하는 행동이다.
- 카카오톡 링크만 눌러도 바로 열리는 흐름이나 알림 메시지 연동이 필요하다.
- 진행 단계별로 `지금 매장이 무엇을 하고 있는지`가 더 친절하게 보여야 한다.

### 5-4. 결제 구조가 아직 서비스 핵심 흐름으로 올라오지 못했다

- 실제 현장에서는 `최종 금액 확정`과 `예약금 확인`이 작업 시작의 핵심 조건이다.
- 카드결제나 간편결제가 없으면 입금 확인 부담이 계속 남는다.
- 예약 주문은 예약금과 잔금 흐름을 나눠 설계해야 한다.

### 5-5. 재구매와 단골화 장치가 부족하다

- 한 번 주문한 고객이 다음에도 다시 쓰게 만드는 장치가 아직 약하다.
- 재주문, 선호 옵션 저장, 최근 주문 불러오기 같은 기능이 필요하다.

## 6. 보완 방향

### 6-1. 서비스 포지셔닝 보완

이 서비스는 `주문 관리 SaaS`보다 `믿고 맡길 수 있는 당일 수산 주문 서비스`로 보여야 한다.

고객 노출 문구는 아래 세 가지를 반복해야 한다.

- 오늘 시세를 먼저 보여준다
- 손질과 전달 과정을 투명하게 안내한다
- 주문 후에도 진행 상황을 직접 확인할 수 있다

### 6-2. 고객 경험 구조 보완

고객이 실제로 체감하는 화면 흐름은 아래 5단계로 단순화하는 것이 좋다.

1. `오늘 주문 가능한가`
2. `어떻게 주문하면 되나`
3. `얼마쯤 나오나`
4. `결제는 어떻게 하나`
5. `지금 어디까지 됐나`

이 기준으로 각 화면의 역할을 더 분명히 나누는 것이 좋다.

## 7. 우선 보완해야 할 항목

### P0. 반드시 먼저 강화할 것

- `결제 구조 명확화`
  - 일반 주문: 최종 금액 확정 후 결제
  - 예약 주문: 예약금 결제 후 진행, 이후 잔금 결제
- `주문 후 상태 조회 진입 단순화`
  - 토큰 입력보다 링크 직접 진입 중심
  - 카카오톡 안내 메시지와 연동 전제 설계
- `주문 마감 시간 표시`
  - 픽업, 퀵, 택배별 마감 시간을 홈과 주문서에 모두 표시
- `예상 금액 범위 안내`
  - 시세 기반 예상 범위, 손질비, 운임비 구조를 고객이 먼저 이해하게 함

### P1. 실사용 만족도를 크게 올리는 항목

- `추천 프리셋`
  - 처음 주문하는 분 추천
  - 회로 바로 드실 분 추천
  - 택배 수령 추천
- `주문 완료 후 알림 자동화`
  - 금액 확정
  - 예약금 확인
  - 손질 시작
  - 출고 완료
- `재주문 기능`
  - 최근 주문 그대로 다시 담기

### P2. 단골화와 객단가를 올리는 항목

- `세트 제안`
  - 회 + 매운탕거리
  - 가족 식사용 추천 조합
- `선물 포장 옵션`
  - 명절/기념일 대응
- `고객 취향 저장`
  - 선호 손질
  - 선호 숙성지
  - 자주 쓰는 수령 방식

## 8. 추천 서비스 구조

### 홈 화면

- 오늘 시세
- 오늘 주문 마감 시간
- 신뢰 문구
- 주문하기
- 주문 조회

### 주문 화면

- 다품목 주문
- 손질/포장 추천
- 수령 방식 분기
- 예상 금액 구조 안내
- 예약 여부 선택

### 결제/확정 단계

- 일반 주문: 최종 금액 안내 후 결제
- 예약 주문: 예약금 결제 후 확보 진행
- 결제 완료 시 자동 상태 전환

### 진행 조회 화면

- 단계별 진행 상태
- 다음 단계 안내
- 결제 상태
- 출고 상태

### 재구매 단계

- 이전 주문 다시 담기
- 자주 주문한 조합 추천
- 최근 받았던 방식 그대로 재주문

## 9. 운영 구조에서 꼭 지켜야 할 원칙

- 미결제 주문은 손질/출고로 자동 전환되지 않아야 한다.
- 예약 주문은 예약금 확인 전까지 확보 진행으로 바뀌지 않아야 한다.
- 택배와 손질 조합 제한은 주문 단계에서 먼저 막아야 한다.
- 고객이 물어보기 전에 `오늘 가능한지`, `어떻게 받는지`, `언제 확인되는지`를 먼저 보여줘야 한다.

## 10. 지금 서비스의 다음 기준 버전

다음 버전은 `카톡 유입을 웹 주문으로 받는 도구`에서 멈추지 않고 아래 수준까지 가야 한다.

- 고객은 시세를 보고 바로 주문할 수 있다.
- 고객은 어렵지 않게 손질과 포장을 고를 수 있다.
- 고객은 금액 확정과 결제를 자연스럽게 이해한다.
- 고객은 주문 후 진행 상황을 별도 문의 없이 확인한다.
- 매장은 입금 확인과 출고 실수를 줄인다.

## 11. 바로 실행할 권장 작업

1. 홈과 주문서에 `주문 마감 시간`, `예상 금액 구조`, `예약 주문 안내`를 강화한다.
2. 예약금/잔금 기준의 결제 구조를 문서와 데이터 모델에 반영한다.
3. 조회 토큰 중심 구조를 `직접 링크 진입` 중심으로 바꾸는 방향을 설계한다.
4. 재주문과 알림 자동화를 백로그 상위로 올린다.
